提交流程

提交 Capstone 專案需要準備以下內容，打包為一篇 Kaggle 文章：

  1. 標題與摘要（150 字以內）：簡述你的 agent 是什麼、解決什麼問題
  2. 說明影片（3–5 分鐘）：展示 agent 運作、講解核心設計思路
  3. 建構理念簡述（500–800 字）：為什麼選擇這個想法、遇到了什麼挑戰、如何克服
  4. 代碼與文檔：
    • 完整的可運行代碼（GitHub repo 或 Kaggle Notebook）
    • 詳細的 README（安裝、運行、功能說明）
    • 代碼註解（特別是複雜邏輯部分）
  5. 測試與評估：展示你如何測試 agent 的品質與安全性

提交檢查清單

  • Agent 功能完整且經過測試
  • 代碼清晰易讀，有適當註解
  • README 包含安裝、使用說明與依賴列表
  • 說明影片清晰、背景音樂適當
  • 文章內容誠懇、邏輯清楚
  • 包含課程中學到的至少 3 個技能點（工具、skills、安全等）
  • Kaggle 文章已發布，連結可訪問

常見問題

我可以單獨開發，還是只能組隊？
兩者皆可。個人開發與團隊開發都被接受。如果是團隊，請在文章中清楚標示各成員的貢獻。
能用付費的 API / 服務嗎？
可以，但請在文章中透明說明成本與使用的服務。評審時會考慮成本效益。
能改進或重用課程中的 codelab 代碼嗎？
完全可以。Capstone 的目的是鞏固學習、實踐知識，而非從零開始。建議在文章中說明你如何擴展或改進了課程內容。
超過截止時間提交會怎樣？
超期提交的文章將不被納入評審。建議留足夠的緩衝時間，避免最後一刻倉促提交。
可以參考或學習他人的代碼嗎？
可以，但必須在文章中清楚標明參考來源與引用。直接複製他人代碼而不標明出處將被視為剽竊，可能被取消資格。
Capstone 專案是否影響課程證書？
不會。課程證書只要求完成 Day 1–5 的內容。Capstone 專案是額外的實踐環節，完成者才能獲得額外的徽章與獎勵。
我不想參與評選，只是想做個人專案，可以嗎？
完全可以。Capstone 是選填的。即使你不參與評選，也歡迎在 Discord 中分享你的成果，與其他學員交流。

準備好了嗎？

如果你還在進行課程，請先完成 Day 1–5，然後回來構思你的 Capstone 想法。

有疑問？加入 Kaggle Discord，與 Google 專家與其他學員交流。
